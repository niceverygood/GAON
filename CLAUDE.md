# 가온 (Gaon) — 상조 플래너용 올인원 엔딩 플래닝 SaaS

> 당신은 가온(Gaon)이라는 신규 SaaS를 처음부터 개발할 시니어 풀스택 엔지니어입니다.
> 이 문서가 프로젝트 전체의 Single Source of Truth입니다. 모든 결정은 여기 맞춰 내리세요.

---

## 🎯 미션

한국 상조 플래너(funeral pre-planning consultants)가 **한 번 써보면 업무 전에는 돌아갈 수 없게** 만드는 B2B SaaS.
죽음이라는 무거운 주제를 **AI가 객관적 수치로 대신 말해주고**, 플래너는 해결책만 제시하게 만드는 도구.

핵심 고객 페인 3개만 해결하면 대박:
1. **"장례는 먼 미래"** 심리 저항 → **엔딩 시나리오** 로 직면화
2. **10년 납입 중 30~50% 해약** → **해약 방지 AI** 로 수십억 손실 방지
3. **신입 플래너가 장례 실전 모름** → **장례 실행 AI 매니저** 로 품질 표준화

## 👥 타깃 사용자

### Primary: 상조 플래너 (B2B2B)
- 한국 주요 상조사(프리드라이프, 보람상조, 교원라이프 등) 소속 영업직원
- 3만~5만명 규모
- 업무: 신규 상조 계약 체결, 기존 고객 유지, 장례 발생 시 지원
- 대부분 40~60대, 디지털 도구 익숙도 중~하 → **UX 쉬워야 함**, **태블릿 최적화 중요**

### Secondary: 상조사 관리자 (super admin)
- 본부장·지점장급
- 성과 대시보드, 해약 리스크 Top N, 플래너 관리

### Tertiary: 고객·유족 (공유 링크 수신자)
- 상담 결과 공유 링크로 접근
- 장례 발생 시 "장례 실행 AI 매니저" 유족용 뷰

## 💎 킬러 3대 기능 (Lock-in 포인트)

### 1️⃣ 엔딩 시나리오 (Endgame Simulator)
- **입력**: 나이·성별·직업·가족구성·자산·기존 상조/보험
- **출력**: 향후 20~30년 인생 엔딩 시뮬레이션
  - 질병 발생 확률·치료비 (AI 기반, 의학 통계 인용)
  - 간병 시작 연령·월 비용·평균 기간
  - 예상 사망 연령(통계 기반)·지역별 장례비
  - 유족 상속세·생계비 갭
- **UX**: 태블릿에서 한 화면에 타임라인으로 시각화
- **설득력**: "막연한 공포 → 구체적 수치"

### 2️⃣ 해약 방지 AI (Retention Guard)
- 기존 가입자 해약 리스크 실시간 점수화
- **스코어 구성**:
  - 미납 경고 (30일 이상 → +30점)
  - 경제 상황 (CODEF 연동 — 소득·대출 변화)
  - 연락 무응답 일수
  - 가족 변동 (이혼·자녀 분가)
  - 타사 상조 가입 탐지
  - 건강 변화 (보비 연동 시)
- **플래너 액션**: "이번 달 해약 위험 Top 10" 자동 제공 → 선제 연락 → AI 화법 추천
- **ROI 증명**: 상조사 1곳 기준 해약 5~10%p 감소 → 수십억 절감

### 3️⃣ 장례 실행 AI 매니저 (Concierge)
- 장례 발생 시 3일간 유족·플래너 옆에 AI 상주
- **유족용**: 사망진단서 발급 → 장례식장 예약 → 화장장 → 부고 문구 → 부의금 정리 → 답례 문자 자동 안내
- **플래너용**: 고객 계약 조회, 옵션별 실시간 견적, 교차 판매 포인트
- **효과**: 신입 플래너도 베테랑급 서비스 → 교육비 절감 + 유족 만족도 ↑

## 🗺 전체 기능 맵

### A. 상담 전 (Pre-call)
- 리드 스코어링 대시보드
- 해약 방지 AI 알림
- 가족관계 CRM (3세대 연결)
- 지역·연령별 사망률·질병률 통계

### B. 상담 중 (During-call) — 태블릿 최적화 필수
- 엔딩 시나리오
- 장례비 vs 상조 보장 갭 시각화
- 가족 단위 통합 분석
- 상속세 예상 계산
- 타사 상조 비교표
- 유언장 기초 초안 (법적 효력은 변호사 안내)

### C. 계약 후 (Post-sale)
- 디지털 멤버십 카드
- 생일·기념일 자동 연락 템플릿
- 납입 이력 시각화

### D. 장례 실행 (On-event)
- 장례 실행 AI 매니저
- 장례식장·화장장 실시간 재고(API 제휴 전까진 평균 DB)
- 부고 문구 생성
- 3일장 타임라인

### E. 사후 서비스 (Post-event)
- 디지털 추모관 (사진·영상)
- 유족 상속 안내
- 기일 알림

### F. 부가 수익 (Optional — Phase 2+)
- 보험 연계 (보비 플랫폼 통합)
- 유언장 공증 중개
- 상속세 신고 대행 (세무사)
- 유품 정리 서비스

보비(BoBi) 프로젝트에서 검증된 스택을 그대로 사용 — 빠른 개발 + 인프라 공유.

### Frontend
- **Next.js 15+ (App Router)** — 서버 컴포넌트 적극 활용
- **TypeScript strict**
- **Tailwind CSS** + **shadcn/ui** 컴포넌트
- **lucide-react** 아이콘
- **태블릿 우선 UI** (min-w 768px 기준 최적화, 모바일도 지원)

### Backend
- **Next.js API Routes** (app/api/)
- **Supabase** — Auth + Postgres + Storage + RLS
- **Supabase RPC 함수** — 성능 크리티컬 쿼리 (엔딩 시나리오·해약 스코어 집계)

### AI
- **Anthropic Claude Sonnet 4.5** (1차 프롬프트: 시나리오 생성, 화법 생성, 유언장 초안)
- **OpenRouter** 폴백 (선택)
- AI 프롬프트 모듈: `lib/ai/prompts.ts` 에서 중앙 관리

### 외부 연동
- **CODEF API** — 주민등록번호 인증, 내보험다보여, 심평원, 건강보험공단 (보비와 공용 계정 가능)
- **카카오** — 로그인 + 알림톡 (플래너 → 고객 안내)
- **PortOne / 토스페이먼츠** — 구독 결제 (보비 결제 인프라 재활용)

### 모니터링·분석
- **Sentry** — 에러
- **PostHog** — 제품 분석 + 퍼널

### 호스팅
- **Vercel** — 자동 배포 (보비와 같은 계정, 별도 프로젝트)

## 🗄 DB 스키마 (초안)

Supabase Postgres, 모든 테이블에 RLS 적용. 자세한 정의는 `scripts/001_initial_schema.sql` 참고.

### 핵심 테이블
- `organizations` — 상조사 (SaaS 구매 주체)
- `planners` — 플래너 사용자
- `clients` — 플래너가 관리하는 고객
- `sangjo_contracts` — 상조 계약
- `ending_scenarios` — 엔딩 시나리오 결과 (AI)
- `retention_scores` — 해약 리스크 스코어 (일일 cron)
- `funeral_events` — 장례 이벤트
- `family_accesses` — 유족 토큰 기반 접근
- `subscriptions` — SaaS 구독

### RLS 정책 요약
- 플래너: 본인 고객·계약만 조회
- 매니저: 소속 조직 전체 조회
- Admin(슈퍼): 전체
- 유족: family_access_token 검증 후 해당 funeral_event만 조회

## 🎨 UX 가이드라인

### 원칙
1. **태블릿 우선** — 플래너는 고객 앞에서 태블릿으로 상담. 가로 모드 기준.
2. **한 화면에 결론** — 스크롤 최소. 핵심 숫자·시각화는 첫 화면에.
3. **죽음 이야기 톤** — 공포 자극 금지. "통계적으로 ~", "평균 ~" 등 객관적 서술.
4. **숫자 강조** — 주요 숫자는 `font-black text-3xl` 수준으로 크게. 보비 스타일.
5. **색상 시스템**:
   - Primary: `indigo-600` (신뢰)
   - Accent: `violet-600` (프리미엄)
   - Warning: `amber-500` (해약 위험)
   - Critical: `rose-600` (장례 이벤트)
   - Calming: `slate-*` (추모·엔딩)

### 접근성
- 모든 텍스트 min 14px
- 버튼 min 44px 높이 (태블릿 터치)
- 색맹 고려: 색상 + 아이콘 + 텍스트 중복

## ⚖ 법률·규제 체크리스트

### 상조업법
- 가온은 **상조 판매 대행 아님** — "영업 지원 도구" 포지셔닝
- 특정 상조사 상품 추천은 회피. 비교만 제공.
- 수수료 기반 중개는 상조사 간 계약으로 처리 (가온은 SaaS 요금만)

### 개인정보보호법
- 고객 정보는 플래너가 **동의 받은** 것만 입력
- 유족 접근 토큰은 **본인 신원 확인 후** 발급
- 민감정보(진료·사망)는 암호화 저장
- CODEF 주민번호 처리는 본인인증 즉시 폐기 (저장 X)

### 법률 서비스
- 유언장·상속세: **"초안"만 생성**, 공증·법적 자문은 변호사·세무사 연결
- "OO을 해야 합니다" 확정 문구 금지 → "OO 검토 권장" 수준

### 표시광고법
- "최고", "1위", "확실히" 같은 확정 표현 금지
- 비교표는 공신력 있는 출처 인용

- **폰트**: Pretendard
- **로고 방향**: 심플, 중심점·선 모티브

## 🚨 Red Line (절대 하지 말 것)

- 특정 상조사·보험사 상품을 직접 판매
- 확정적 사망 연령 제시 ("당신은 78세에 죽습니다" 금지)
- 공포 마케팅 문구
- 보비 고객·분석 데이터 무단 사용
- 법률·세무·의료 확정 자문 제공
- RLS 우회 서버 사이드 쿼리 남용
